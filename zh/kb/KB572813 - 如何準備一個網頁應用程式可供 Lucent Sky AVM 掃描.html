介紹

Lucent Sky AVM 在偵測弱點之前，需要先確認應用程式是可以被建置及編譯的。這篇文章將示範如何在將 Web 應用程式的程式碼上傳至 Lucent Sky AVM 掃描之前，發現建置及編譯的問題，同時也概述解決這些問題的步驟。

更多資訊

有許多整合開發環境（integrated development environment，IDE）可以建置網頁應用程式－－包含其類別（class）和網頁（webpage），並識別建置和編譯的問題，像是不正確的語法或缺少檔案及程式庫（library）。使用這個方法的好處是這可以同時確定應用程式的結構符合 Lucent Sky AVM 所支援的格式。描述此內容的步驟如下：

.NET

  1. 使用 Visual Studio 開啟網頁應用程式或網頁 - 在 [檔案] 目錄下，選擇 [開啟]、接著依你的應用程式類型選擇 [專案/方案] 或 [網站] 選項。接著在視窗中選擇應用程式的根目錄或專案檔後，點選 [開啟]。
  2. 建置網頁應用程式 – 若這是一個網站，請跳過這個步驟。在 ［方案總管］ 中，以滑鼠右鍵點擊該網頁應用程式，接著在目錄中選擇 [建置方案]。此時要確認在建置過程中沒有錯誤產生。如果有發生缺少檔案的錯誤，請選擇 2(a) 或 2(b) 其一的方法來排除：
    1. 將缺少的 User Control 或檔案放置到錯誤訊息中提示缺少的位置。
    2. 若已經不再使用該 Web Form，請移除其檔案（.aspx）。
    若為其他錯誤，請依照應用程式的需求解析並排除錯誤，重複這個步驟直到不再有錯誤訊息出現。
  3. 發行網頁應用程式 - 在［方案總管］，以滑鼠右鍵點擊網頁應用程式，接著在目錄中選擇 [發行網站]。在 [發行網站] 對話方塊中，選擇你要發行網站的位置，接著按下[確定]。
  4. 確認網站可被發行 - 確認發行過程中沒有錯誤。一旦發行成功，狀態列將出現發行成功的訊息。如果有發生缺少檔案的錯誤，請選擇 4(a) 或 4(b) 其一的方法來排除：
    1. 將缺少的 User Control 或 file 放置到錯誤訊息中提示缺少的位置。
    2. 若已經不再使用該 Web Form，請移除其檔案（.aspx）。
    若為其他錯誤，請依照應用程式的需求解析並排除錯誤，重複這個步驟直到不再有錯誤訊息出現。
  5. 準備掃描使用的應用程式檔案 - 在 [檔案管理員] 中瀏覽應用程式的根目錄，選取全部的網頁應用程式資料夾及檔案，以滑鼠右鍵點選選擇的檔案，將滑鼠移到 [傳送到(N)] 標籤，並選擇 [壓縮的(Zipped) 資料夾]。
  6. 開始掃描應用程式 - 在 Lucent Sky AVM Web UI 中，建立一個新的掃描並上個步驟中所建立的壓縮檔案作為程式碼壓縮檔.

Java

  1. 確認網頁應用程式可被建構 - 在 Project Explorer 視窗中，在專案上按一下滑鼠右鍵並選擇 Build Project。再來，在專案上按一下滑鼠右鍵並選擇 Validate。依需要修改程式碼，直到在編譯和驗證時沒有錯誤產生。
  2. 確認所有被參考的 JAR（.jar）檔案都會被匯出 - 在 Project Explorer 視窗中，在專案上按一下滑鼠右鍵並選擇 Properties，再選擇 Java Build Path，接著選擇Order and Export 頁籤。確認所有被這個應用程式參考的 JAR 檔案都有被選取，以確保這些函式庫在下個步驟會一起被匯出。
  3. 匯出 WAR（.war）檔案 - 在 Project Explorer 視窗中，在專案上按一下滑鼠右鍵並選擇 Export。勾選 Export source files，再為專案取一個名稱以及匯出的目的位置。
  4. 確認匯出的 WAR（.war）檔案 - 將匯出的 .war 檔案的附檔名改為 .zip。開啟這個 .zip 檔案並確認 WEB-INF\lib 目錄中包含這個專案的 JAR 參照，以及 WEB-INF\classes 目錄中包含 Java 程式碼檔案（.java），而不僅是 .class 檔案。
  5. 開始掃描應用程式 - 在 Lucent Sky AVM Web UI 中，建立一個新的掃描並上個步驟中所建立的壓縮檔案作為程式碼壓縮檔.
