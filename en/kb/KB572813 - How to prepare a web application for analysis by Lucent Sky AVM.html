Introduction

Lucent Sky AVM requires an application to be buildable and compilable before it be can analyzed. This article demonstrates how to identify build and compilation errors in a web application, before submitting it to Lucent Sky AVM for analysis. It also outlines steps that might help resolve these errors.

More Information

Many integrated development environments (IDEs) are capable of building a web application, including its classes and webpages, and identify any build and compilation error, such as invalid syntax or missing files and libraries. A benefit of taking this approach is that it will also ensure the application is in a structure that's supported by Lucent Sky AVM. The steps below outline this approach:

.NET

  1. Open the web application or website in Visual Studio - In the File menu, select Open, then select either Project/Solution or Web Site. In the dialog box, choose the root directory or project file of the legacy website, then click Open.
  2. Build the web application– If it's a website, skip this step. In Solution Explorer, right-click on the web application, then select Build Solution in the contextual menu. Verify that no error occurred during the build process. If file not found errors occur during the publication, follow either step 2(a) or 2(b):
    1. Locate the missing User Control or file, and place it in the location indicated by the error message.
    2. Remove the Web Form (.aspx) that's causing the error, if this's no longer used. For other errors, resolving them according to the application requirement. Repeat this step until there're no more errors.
  3. Publish the web application - In Solution Explorer, right-click on the web application project, then select Publish... in the contextual menu. For Target Location, choose a temporary location to publish the legacy website to. Then click OK.
  4. Verify the website is publishable - Verify that no error occurred during the publication process. Once the publication has completed successfully, the status bar should display the message Publish succeeded. If file not found errors occur during the publication, follow either step 4(a) or 4(b):
    1. Locate the missing User Control or file, and place it in the location indicated by the error message.
    2. Remove the Web Form (.aspx) that's causing the error, if this's no longer used. For other errors, resolving them according to the application requirement. Repeat this step until there're no more errors. Publish the website again by starting at step 3.
  5. Package the web application for analysis - In File Explorer, browse to the root directory of the website. Select all files, right-click, select Send to, then select Compressed (zipped) folder.
  6. Start a scan of the web application or website - In Lucent Sky AVM Web UI, create a new scan and use the zip file created in step 5 as Source Code Archive.

Java

  1. Verify the web application is buildable - In the Project Explorer window, right-click on the project and select Build Project. Then, right-click on the project and select Validate. Make revisions to the source code as necessary to correct for any errors in compilation or validation that occur.
  2. Verify all referenced JAR files will be exported - In the Project Explorer window, right-click on the project, and select Properties, then select Java Build Path, then the Order and Export tab. Ensure that every JAR dependency referenced by the application is selected, so that these libraries will be placed in the WEB-INF\lib directory in the WAR file created in the next step.
  3. Export the .war file - In the Project Explorer window, right-click on the project and select Export. Check the Export source files box, then select a name for the project and a destination to which it will be saved.
  4. Verify the exported .war file - Rename the extension of the WAR file from .war to .zip. Open the .zip file and verify that the directory WEB-INF\lib contains the project's JAR dependencies, and that the directory WEB-INF\classes contains Java source files, not just .class files.
  5. Start a scan of the web application - In Lucent Sky AVM Web UI, create a new scan and use the .zip file created in step 4 as Source Code Archive.
